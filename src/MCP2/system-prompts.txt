# MCP2 ‚Äî Enhanced Development Tools

MCP2 is a unified MCP server providing 69 tools for file system operations, code editing, database access, document processing, HTTP requests, compression, image viewing, and shell command execution. It runs as a single .NET Framework 4.8 console application communicating via JSON-RPC 2.0 over stdin/stdout.

## Configuration

All settings are in `mcp-config.json` next to the executable:

- `mysql_connection_string` ‚Äî MySQL database connection string.
- `gc_memory_threshold_mb` ‚Äî Working set threshold (MB) to trigger full GC. Default: 150.
- `debug_logging` ‚Äî Enable debug log file (`mcp_debug.log`). Default: false.
- `backup_directory` ‚Äî Custom backup folder. Default: `./backups` next to exe.

## Tool Reference (60 tools)

### File Operations (13 tools)

**read_file** ‚Äî Read the complete raw content of a file without line numbers.
- `path` (string, required): Full path to the file

**read_file_lines** ‚Äî Read file content with line numbers prefixed to each line. Supports optional line range.
- `path` (string, required): Full path to the file
- `start_line` (integer, optional): Starting line number (1-based). If omitted, reads from beginning.
- `end_line` (integer, optional): Ending line number (1-based). If omitted, reads to end of file.

**read_line_range** ‚Äî Read only a specific range of lines from a file.
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based, inclusive)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- Out-of-range behavior: If `start_line` > file length, returns an info message with no data. If `end_line` > file length, silently clamps to the last line and returns available lines.

**count_lines** ‚Äî Get the total number of lines in a file. Returns -1 for binary files.
- `path` (string, required): Full path to the file

**find_pattern** ‚Äî Search for a pattern in a file and return all matching lines with their line numbers.
- `path` (string, required): Full path to the file
- `pattern` (string, required): Text pattern to search for
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `max_results` (integer): Maximum number of matches to return (0 = all). Default: 0

**find_all_occurrences** ‚Äî Find all instances of text with surrounding context lines. Use this to identify which occurrence to edit when there are multiple matches.
- `path` (string, required): Full path to the file
- `pattern` (string, required): Text pattern to search for
- `context_lines` (integer): Number of lines before/after each match. Default: 2
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true

**write_file** ‚Äî Create a new file or overwrite an existing file with the specified content. Uses UTF-8 with BOM by default (ensures correct rendering of non-ASCII characters like Chinese text in browsers and editors). Returns file size and line count.
- `path` (string, required): Full path of the file to write
- `content` (string, required): Content to write to the file
- `encoding` (string): File encoding ‚Äî "utf8-bom" (default, recommended for web files with non-ASCII characters) or "utf8" (without BOM)

**append_to_file** ‚Äî Append content to the end of an existing file, or create a new file if it doesn't exist.
- `path` (string, required): Full path of the file
- `content` (string, required): Content to append

**copy_file** ‚Äî Copy a file to a new location.
- `source_path` (string, required): Full path of the source file
- `destination_path` (string, required): Full path of the destination file
- `overwrite` (boolean): If true, overwrite existing destination file. Default: false

**move_file** ‚Äî Move or rename a file.
- `source_path` (string, required): Full path of the source file
- `destination_path` (string, required): Full path of the destination file

**delete_file** ‚Äî Delete a file. Use with caution!
- `path` (string, required): Full path of the file to delete

**file_exists** ‚Äî Check if a file exists. Returns 'true' or 'false'.
- `path` (string, required): Full path to the file

**get_file_info** ‚Äî Get metadata about a file (size, dates, attributes).
- `path` (string, required): Full path to the file

### File Editing (12 tools)

WARNING: Line numbers change after edits! For multiple edits to the same file, either use batch_edit (auto-sorts bottom-up), edit from bottom to top manually, or re-read the file between edits.

**replace** ‚Äî Replace a single line with new content.
- `path` (string, required): Full path to the file
- `line_number` (integer, required): Line number to replace (1-based)
- `new_content` (string, required): New content for the line
- `create_backup` (boolean): Create backup before editing. Default: true
- Out-of-range behavior: If `line_number` > file length, makes no changes and returns an error message.

**replace_line_range** ‚Äî Replace a range of lines (start to end, inclusive) with new content. The new content can have more or fewer lines than the original range. This is the primary tool for replacing blocks of code.
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based, inclusive)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- `new_content` (string, required): New content to replace the line range with
- `create_backup` (boolean): Create backup before editing. Default: true
- Out-of-range behavior: If `start_line` > file length, makes no changes and returns an error message. If `end_line` > file length, silently clamps to the last line and replaces from `start_line` to actual file end.

**insert_at** ‚Äî Insert new content before a specific line.
- `path` (string, required): Full path to the file
- `line_number` (integer, required): Line number where content will be inserted (1-based)
- `content` (string, required): Content to insert
- `create_backup` (boolean): Create backup before editing. Default: true

**insert_after** ‚Äî Insert new content after a specific line.
- `path` (string, required): Full path to the file
- `line_number` (integer, required): Line number after which content will be inserted (1-based)
- `content` (string, required): Content to insert
- `create_backup` (boolean): Create backup before editing. Default: true

**delete** ‚Äî Delete a range of lines from a file.
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- `create_backup` (boolean): Create backup before editing. Default: true
- Out-of-range behavior: If `start_line` > file length, makes no changes and returns an error message. If `end_line` > file length, silently clamps to the last line and deletes from `start_line` to actual file end.

**edit_nth_occurrence** ‚Äî Replace the Nth occurrence of text in a file. Supports both single-line text and multi-line blocks (content can span multiple lines). Use find_all_occurrences first to identify which occurrence number you need.
- `path` (string, required): Full path to the file
- `find_text` (string, required): Text or multi-line block to find. Must match file content exactly (including whitespace and indentation).
- `occurrence` (integer, required): Which occurrence to replace (1-based)
- `replace_text` (string, required): Replacement text or multi-line block. Can be more or fewer lines than find_text.
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `create_backup` (boolean): Create backup before editing. Default: true

**batch_edit** ‚Äî Perform multiple edits across one or more files in a single operation. Edits are automatically sorted bottom-up per file to preserve line numbers. Supports operations: replace, insert_at, insert_after, delete.
- `edits` (array, required): Array of edit operations. Each object: {file, type, line or start_line, end_line (optional), content}. Use `line` for single-line targets (insert_at, insert_after). Use `start_line`/`end_line` for ranges (replace, delete). Both `line` and `start_line` are accepted interchangeably.
- `create_backup` (boolean): Create backup for each file before editing. Default: true

**replace_in_line_range** ‚Äî Find and replace text within a specific range of lines (text pattern match within a line range, not a full line replacement).
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based, inclusive)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- `find_text` (string, required): Text to find
- `replace_text` (string, required): Text to replace with
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: false
- `create_backup` (boolean): Create backup before editing. Default: true

**replace_regex** ‚Äî Find and replace using regular expressions. Supports $1, $2 capture groups.
- `path` (string, required): Full path to the file
- `regex_pattern` (string, required): Regular expression pattern
- `replacement` (string, required): Replacement string (supports $1, $2 capture groups)
- `create_backup` (boolean): Create backup before editing. Default: true

**replace_all** ‚Äî Replace all occurrences of text in a file. Supports both single-line text and multi-line blocks (content can span multiple lines). Reports the number of replacements made. Use replace_first with must_be_unique=true when you want to ensure exactly one match exists before replacing.
- `path` (string, required): Full path to the file
- `find_text` (string, required): Text or multi-line block to find. Must match file content exactly (including whitespace and indentation).
- `replace_text` (string, required): Replacement text or multi-line block. Can be more or fewer lines than find_text.
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `create_backup` (boolean): Create backup before editing. Default: true

**replace_first** ‚Äî Replace the first occurrence of text in a file. Supports both single-line text and multi-line blocks (content can span multiple lines). When must_be_unique is true (default), the find_text must appear exactly once in the file or the operation is rejected ‚Äî this prevents accidental edits when the match is ambiguous. This is the safest content-based replace tool (equivalent to Anthropic's str_replace pattern).
- `path` (string, required): Full path to the file
- `find_text` (string, required): Text or multi-line block to find. Must match file content exactly (including whitespace and indentation). For multi-line blocks, include the full block as it appears in the file.
- `replace_text` (string, required): Replacement text or multi-line block. Can be more or fewer lines than find_text.
- `must_be_unique` (boolean): If true (default), find_text must appear exactly once in the file ‚Äî rejects if 0 or 2+ matches found. Set to false to replace just the first occurrence even when duplicates exist. Default: true
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `create_backup` (boolean): Create backup before editing. Default: true

### Shell Tools (1 tool)

**run_command** ‚Äî Execute an external program (cmd, powershell, or any executable) and return the output. stdout and stderr are captured directly in C# via stream redirection (not shell redirection), so output is always returned reliably. For single-line commands, pass program + parameters. For multi-line PowerShell scripts, write a .ps1 file first and pass it via script_path. ESCAPING NOTE: For PowerShell, the parameters string is wrapped in double quotes and passed via -Command. Only inner double quotes are escaped (doubled). If your command contains $variables, backticks, or complex quoting, write a .ps1 file and use script_path instead ‚Äî this avoids all escaping issues.
- `program` (string, required): Filename or full path to the executable (e.g., "powershell", "cmd", "ipconfig", "pnputil")
- `parameters` (string): Command-line arguments for single-line execution. For PowerShell, do NOT include `-Command` or `-File` ‚Äî just pass the command string directly. IMPORTANT: Only inner double quotes are escaped. If your command uses $variables, backticks, or complex quoting, use script_path instead.
- `script_path` (string): Full path to a script file (e.g., .ps1) to execute. Use this for multi-line PowerShell scripts instead of inline parameters. `-NoProfile -ExecutionPolicy Bypass` is applied automatically. This is the PREFERRED approach for any PowerShell command containing $variables, backticks, single quotes, or complex expressions ‚Äî it bypasses all escaping issues.
- `report_log` (string): Optional path to save a copy of the captured output. If omitted, output is still fully returned ‚Äî no log file needed. Auto-generated temp files are deleted after reading.
- `wait_for_exit` (boolean): Wait for the process to finish before returning. Default: true. Set to false for fire-and-forget processes.
- `timeout_seconds` (integer): Maximum seconds to wait before killing the process. Default: 30. Increase for long-running operations (e.g., installs, builds).

## Backups

- File editing tools create timestamped backups by default (`.bak` files)
- Use `undo_last_edit` to quickly revert changes
- Use `clear_backups` to manage backup storage
- Backup location configurable via `backup_directory` in config
