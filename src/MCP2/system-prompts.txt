# MCP2 — Enhanced Development Tools

MCP2 is a unified MCP server providing 90+ tools for file system operations, code editing, database access, SSH remote access, project building, HTTP requests, compression, image viewing, and shell command execution. It runs as a single .NET Framework 4.8 console application communicating via JSON-RPC 2.0 over stdin/stdout.

## Configuration

All settings are in `mcp-config.json` next to the executable:

- `mysql_connection_string` — MySQL database connection string.
- `gc_memory_threshold_mb` — Working set threshold (MB) to trigger full GC. Default: 150.
- `debug_logging` — Enable debug log file (`mcp_debug.log`). Default: false.
- `backup_directory` — Custom backup folder. Default: `./backups` next to exe.
- `ssh_profiles` — Named SSH connection profiles (see SSH Tools section).

## Tool Reference

### File Operations (13 tools)

**read_file** — Read the complete raw content of a file without line numbers.
- `path` (string, required): Full path to the file

**read_file_lines** — Read file content with line numbers prefixed to each line. Supports optional line range.
- `path` (string, required): Full path to the file
- `start_line` (integer, optional): Starting line number (1-based). If omitted, reads from beginning.
- `end_line` (integer, optional): Ending line number (1-based). If omitted, reads to end of file.

**read_line_range** — Read only a specific range of lines from a file.
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based, inclusive)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- Out-of-range behavior: If `start_line` > file length, returns an info message with no data. If `end_line` > file length, silently clamps to the last line and returns available lines.

**count_lines** — Get the total number of lines in a file. Returns -1 for binary files.
- `path` (string, required): Full path to the file

**find_pattern** — Search for a pattern in a file and return all matching lines with their line numbers.
- `path` (string, required): Full path to the file
- `pattern` (string, required): Text pattern to search for
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `max_results` (integer): Maximum number of matches to return (0 = all). Default: 0

**find_all_occurrences** — Find all instances of text with surrounding context lines. Use this to identify which occurrence to edit when there are multiple matches.
- `path` (string, required): Full path to the file
- `pattern` (string, required): Text pattern to search for
- `context_lines` (integer): Number of lines before/after each match. Default: 2
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true

**write_file** — Create a new file or overwrite an existing file with the specified content. Uses UTF-8 with BOM by default (ensures correct rendering of non-ASCII characters like Chinese text in browsers and editors). Returns file size and line count.
- `path` (string, required): Full path of the file to write
- `content` (string, required): Content to write to the file
- `encoding` (string): File encoding — "utf8-bom" (default, recommended for web files with non-ASCII characters) or "utf8" (without BOM)

**append_to_file** — Append content to the end of an existing file, or create a new file if it doesn't exist.
- `path` (string, required): Full path of the file
- `content` (string, required): Content to append

**copy_file** — Copy a file to a new location.
- `source_path` (string, required): Full path of the source file
- `destination_path` (string, required): Full path of the destination file
- `overwrite` (boolean): If true, overwrite existing destination file. Default: false

**move_file** — Move or rename a file.
- `source_path` (string, required): Full path of the source file
- `destination_path` (string, required): Full path of the destination file

**delete_file** — Delete a file. Use with caution!
- `path` (string, required): Full path of the file to delete

**file_exists** — Check if a file exists. Returns 'true' or 'false'.
- `path` (string, required): Full path to the file

**get_file_info** — Get metadata about a file (size, dates, attributes).
- `path` (string, required): Full path to the file

### File Editing (12 tools)

WARNING: Line numbers change after edits! For multiple edits to the same file, either use batch_edit (auto-sorts bottom-up), edit from bottom to top manually, or re-read the file between edits.

**replace** — Replace a single line with new content.
- `path` (string, required): Full path to the file
- `line_number` (integer, required): Line number to replace (1-based)
- `new_content` (string, required): New content for the line
- `create_backup` (boolean): Create backup before editing. Default: true
- Out-of-range behavior: If `line_number` > file length, makes no changes and returns an error message.

**replace_line_range** — Replace a range of lines (start to end, inclusive) with new content. The new content can have more or fewer lines than the original range. This is the primary tool for replacing blocks of code.
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based, inclusive)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- `new_content` (string, required): New content to replace the line range with
- `create_backup` (boolean): Create backup before editing. Default: true
- Out-of-range behavior: If `start_line` > file length, makes no changes and returns an error message. If `end_line` > file length, silently clamps to the last line and replaces from `start_line` to actual file end.

**insert_at** — Insert new content before a specific line.
- `path` (string, required): Full path to the file
- `line_number` (integer, required): Line number where content will be inserted (1-based)
- `content` (string, required): Content to insert
- `create_backup` (boolean): Create backup before editing. Default: true

**insert_after** — Insert new content after a specific line.
- `path` (string, required): Full path to the file
- `line_number` (integer, required): Line number after which content will be inserted (1-based)
- `content` (string, required): Content to insert
- `create_backup` (boolean): Create backup before editing. Default: true

**delete** — Delete a range of lines from a file.
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- `create_backup` (boolean): Create backup before editing. Default: true
- Out-of-range behavior: If `start_line` > file length, makes no changes and returns an error message. If `end_line` > file length, silently clamps to the last line and deletes from `start_line` to actual file end.

**edit_nth_occurrence** — Replace the Nth occurrence of text in a file. Supports both single-line text and multi-line blocks (content can span multiple lines). Use find_all_occurrences first to identify which occurrence number you need.
- `path` (string, required): Full path to the file
- `find_text` (string, required): Text or multi-line block to find. Must match file content exactly (including whitespace and indentation).
- `occurrence` (integer, required): Which occurrence to replace (1-based)
- `replace_text` (string, required): Replacement text or multi-line block. Can be more or fewer lines than find_text.
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `create_backup` (boolean): Create backup before editing. Default: true

**batch_edit** — Perform multiple edits across one or more files in a single operation. Edits are automatically sorted bottom-up per file to preserve line numbers. Supports operations: replace, insert_at, insert_after, delete.
- `edits` (array, required): Array of edit operations. Each object: {file, type, line or start_line, end_line (optional), content}. Use `line` for single-line targets (insert_at, insert_after). Use `start_line`/`end_line` for ranges (replace, delete). Both `line` and `start_line` are accepted interchangeably.
- `create_backup` (boolean): Create backup for each file before editing. Default: true

**replace_in_line_range** — Find and replace text within a specific range of lines (text pattern match within a line range, not a full line replacement).
- `path` (string, required): Full path to the file
- `start_line` (integer, required): Starting line number (1-based, inclusive)
- `end_line` (integer, required): Ending line number (1-based, inclusive)
- `find_text` (string, required): Text to find
- `replace_text` (string, required): Text to replace with
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: false
- `create_backup` (boolean): Create backup before editing. Default: true

**replace_regex** — Find and replace using regular expressions. Supports $1, $2 capture groups.
- `path` (string, required): Full path to the file
- `regex_pattern` (string, required): Regular expression pattern
- `replacement` (string, required): Replacement string (supports $1, $2 capture groups)
- `create_backup` (boolean): Create backup before editing. Default: true

**replace_all** — Replace all occurrences of text in a file. Supports both single-line text and multi-line blocks (content can span multiple lines). Reports the number of replacements made. Use replace_first with must_be_unique=true when you want to ensure exactly one match exists before replacing.
- `path` (string, required): Full path to the file
- `find_text` (string, required): Text or multi-line block to find. Must match file content exactly (including whitespace and indentation).
- `replace_text` (string, required): Replacement text or multi-line block. Can be more or fewer lines than find_text.
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `create_backup` (boolean): Create backup before editing. Default: true

**replace_first** — Replace the first occurrence of text in a file. Supports both single-line text and multi-line blocks (content can span multiple lines). When must_be_unique is true (default), the find_text must appear exactly once in the file or the operation is rejected — this prevents accidental edits when the match is ambiguous. This is the safest content-based replace tool (equivalent to Anthropic's str_replace pattern).
- `path` (string, required): Full path to the file
- `find_text` (string, required): Text or multi-line block to find. Must match file content exactly (including whitespace and indentation). For multi-line blocks, include the full block as it appears in the file.
- `replace_text` (string, required): Replacement text or multi-line block. Can be more or fewer lines than find_text.
- `must_be_unique` (boolean): If true (default), find_text must appear exactly once in the file — rejects if 0 or 2+ matches found. Set to false to replace just the first occurrence even when duplicates exist. Default: true
- `case_sensitive` (boolean): Whether search is case-sensitive. Default: true
- `create_backup` (boolean): Create backup before editing. Default: true

### Shell Tools (1 tool)

**run_command** — Execute an external program (cmd, powershell, or any executable) and return captured stdout + stderr. Output is captured via C# stream redirection, so it is always returned reliably regardless of shell.

Three ways to run PowerShell (in order of preference):

1. **`script`** (PREFERRED) — Paste inline script content directly, multi-line supported. Auto-written to a temp .ps1 file, executed with `-NoProfile -ExecutionPolicy Bypass`, temp file deleted after execution. No escaping needed. Use this for anything with `$variables`, backticks, pipelines, loops, or multi-line logic.
2. **`script_path`** — Point to an existing .ps1 file on disk. Same execution flags applied automatically. Use when the script already exists as a file.
3. **`parameters`** — For trivial single-line commands only (e.g., `Get-Date`, `ls`). Wrapped in `-Command "..."` with inner double quotes escaped. Avoid for anything containing `$variables` or complex quoting.

For non-PowerShell programs, use `program` + `parameters` (e.g., `program: "ipconfig"`, `parameters: "/all"`).

Parameters:
- `program` (string, required): Filename or full path to the executable (e.g., "powershell", "cmd", "ipconfig", "mysqld.exe")
- `parameters` (string): Command-line arguments for single-line execution. For PowerShell, do NOT include `-Command` or `-File` — just the command string.
- `script` (string): Inline script content (multi-line supported). For PowerShell: auto-written to a temp .ps1 file, executed, then deleted. For cmd: auto-written to a temp .bat file, then deleted. No escaping needed — just paste your script code directly.
- `script_path` (string): Full path to an existing script file (e.g., .ps1) on disk. Use `script` instead if you want to pass the script content inline.
- `report_log` (string): Optional path to save a copy of the captured output. If omitted, output is still fully returned — no log file needed.
- `working_directory` (string): Working directory for the process. If not specified, inherits from the MCP2 server process.
- `wait_for_exit` (boolean): Wait for the process to finish before returning. Default: true. Set to false for fire-and-forget processes.
- `timeout_seconds` (integer): Maximum seconds to wait before killing the process. Default: 30. Increase for long-running operations (e.g., builds, installs, database init).

Timeout behavior: If the process exceeds `timeout_seconds`, it is killed and the response includes any partial stdout/stderr captured before the timeout (capped at 20KB each). This ensures you always get diagnostic output even when a process hangs.

Usage examples:

```
// Simple: run an executable directly
program: "ipconfig"
parameters: "/all"

// PowerShell one-liner via parameters
program: "powershell"
parameters: "Get-Service MySQL_Main"

// PowerShell multi-line via script (PREFERRED)
program: "powershell"
script: |
  $svc = Get-Service MySQL_Main -ErrorAction SilentlyContinue
  if ($svc.Status -eq 'Running') {
      Write-Host "MySQL is running (PID: $($svc.ServiceHandle))"
  } else {
      Write-Host "MySQL is stopped"
      Start-Service MySQL_Main
      Write-Host "Started MySQL"
  }
timeout_seconds: 15

// Run an existing .ps1 file
program: "powershell"
script_path: "D:\\scripts\\deploy.ps1"
timeout_seconds: 120

// CMD batch
program: "cmd"
parameters: "dir /s /b *.cs"
working_directory: "D:\\projects\\myapp"

// Direct executable
program: "D:\\mysql\\server_main\\bin\\mysql.exe"
parameters: "-u root -p1234 -e \"SELECT VERSION();\""
```

### Build Tools (1 tool)

**msbuild** — Build .NET Framework projects (.csproj, .sln, .slnx) using MSBuild. Auto-discovers MSBuild.exe from the latest installed Visual Studio — no configuration needed. Scans `C:\Program Files\Microsoft Visual Studio\{version}\{edition}\MSBuild\Current\Bin\MSBuild.exe`, picks the highest version, and caches the path for the process lifetime.
- `project` (string, required): Full path to the .csproj, .sln, or .slnx file to build
- `target` (string enum): MSBuild target — "Build" (default), "Rebuild", "Clean", "Restore"
- `configuration` (string enum): Build configuration — "Debug" (default), "Release"
- `verbosity` (string enum): Output verbosity — "quiet", "minimal" (default), "normal", "detailed", "diagnostic"
- `timeout_seconds` (integer): Maximum seconds to wait for the build. Default: 120

Usage examples:

```
// Build a project (Debug)
project: "D:\\projects\\MyApp\\MyApp.csproj"

// Build a solution (Release)
project: "D:\\projects\\MyApp\\MyApp.sln"
configuration: "Release"

// Rebuild (clean + build)
project: "D:\\projects\\MyApp\\MyApp.csproj"
target: "Rebuild"

// NuGet restore
project: "D:\\projects\\MyApp\\MyApp.csproj"
target: "Restore"
```

### SSH Tools (5 tools)

SSH tools use named profiles defined in `mcp-config.json` under `ssh_profiles`. Credentials are stored in the config file — never passed as tool parameters.

**Profile configuration** in `mcp-config.json`:

```json
{
  "ssh_profiles": {
    "myserver": {
      "host": "192.168.1.100",
      "port": 22,
      "username": "admin",
      "password": "your-password"
    },
    "myvps": {
      "host": "vps.example.com",
      "port": 22,
      "username": "root",
      "private_key_path": "C:\\Users\\you\\.ssh\\id_rsa",
      "passphrase": ""
    }
  }
}
```

#### Interactive Shell (3 tools)

For interactive, stateful remote sessions. The shell is persistent — environment variables, working directory (`cd`), and other state carry over between `ssh_send` calls.

**ssh_open** — Open a persistent SSH connection using a named profile. The profile name becomes the session identifier for subsequent ssh_send and ssh_close calls.
- `profile` (string, required): SSH profile name from mcp-config.json

**ssh_send** — Send a command to an open SSH session and return the output. The session must be opened first with ssh_open.
- `command` (string, required): Shell command to execute on the remote server
- `profile` (string): Which session (profile name) to send to. Default: "default"
- `timeout_ms` (integer): Maximum milliseconds to wait for output. Default: 10000. Increase for slow commands. Max: 120000.

**ssh_close** — Close an open SSH session and release resources.
- `profile` (string): Which session to close. Default: "default". Use "all" to close every open session.

Usage examples:

```
// Open connection
ssh_open: profile: "myserver"

// Run commands (state persists between calls)
ssh_send: profile: "myserver", command: "cd /var/www && ls -la"
ssh_send: profile: "myserver", command: "pwd"  // still in /var/www

// Close when done
ssh_close: profile: "myserver"
```

#### File Transfer (2 tools)

For uploading and downloading files/folders via SFTP. These are one-shot operations — they open a connection, transfer, and close automatically. They do NOT require ssh_open.

**ssh_upload** — Upload local files and/or folders to a remote server via SFTP. Accepts a mix of file paths and directory paths — directories are uploaded recursively. Remote directories are created as needed.
- `profile` (string, required): SSH profile name from mcp-config.json
- `paths` (array, required): Array of local file or folder paths to upload
- `destination` (string, required): Remote destination folder path (e.g., "/home/user/project/")
- `overwrite` (boolean): Overwrite existing remote files. Default: true

**ssh_download** — Download remote files and/or folders from a server via SFTP. Accepts a mix of remote file and directory paths — directories are downloaded recursively. Local directories are created as needed.
- `profile` (string, required): SSH profile name from mcp-config.json
- `paths` (array, required): Array of remote file or folder paths to download
- `destination` (string, required): Local destination folder path (e.g., "D:\\downloads\\")
- `overwrite` (boolean): Overwrite existing local files. Default: true

Usage examples:

```
// Upload files and folders to remote server
ssh_upload:
  profile: "myserver"
  paths: ["D:\\project\\src\\", "D:\\project\\web.config"]
  destination: "/var/www/mysite/"

// Download from remote server
ssh_download:
  profile: "myserver"
  paths: ["/var/log/myapp.log", "/var/www/mysite/uploads/"]
  destination: "D:\\backups\\server\\"
```

## Backups

- File editing tools create timestamped backups by default (`.bak` files)
- Use `undo_last_edit` to quickly revert changes
- Use `clear_backups` to manage backup storage
- Backup location configurable via `backup_directory` in config
